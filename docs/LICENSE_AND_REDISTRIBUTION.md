# License And Redistribution

## Current posture

- Keep existing Wonder Unit license files and notices intact.
- Do not change packaging identity, publisher metadata, updater targets, or signing assumptions without explicit review.
- Do not remove `build/license_en.txt` or related distribution assets.

## Contributor expectations

- Code contributions should remain compatible with the repository's current licensing and CLA flow.
- Third-party assets or fonts must have clear redistribution terms before merge.
- Cherry-picked fork changes should be reviewed for attribution, license compatibility, and removed assets.

## Release checklist

- Confirm license files are still packaged.
- Confirm repository/author/app identity changes were not introduced accidentally.
- Confirm bundled dependencies and assets are still distributable.
- Record any fork-specific branding or distribution decisions in release notes.
